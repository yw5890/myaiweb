유저 플로우를 추론하기 위해
유저의 행동 정의

테이블을 만들어둠.

CRUD
- create: 데이터 추가하기 (회원가입/게시물작성/댓글작성)
- read: 데이터 읽기 (조회하기->게시물 목록 가져오기, 게시물 내용 가져오기)
- update: 데이터 수정하기(닉네임변경, 게시물 내용 수정, 댓글 수정)
- delete: 데이터 제거하기 (게시물삭제, 댓글 삭제, 회원탈퇴)


-------------------------------------------------------------------
5. 데이터베이스 테이블 설계
-------------------------------------------------------------------

[1] users (회원 테이블)
  - id          : UUID        PK  고유 식별자
  - email       : VARCHAR         아이디(이메일)
  - password    : VARCHAR         비밀번호 (암호화)
  - nickname    : VARCHAR         닉네임
  - phone       : VARCHAR         전화번호
  - created_at  : TIMESTAMP       가입일
  ※ 소셜 로그인 지원 시 provider, provider_id 컬럼 추가 필요

[2] posts (게시물 테이블)
  - id           : UUID        PK  고유 식별자
  - user_id      : UUID        FK  작성자 (users.id 참조)
  - title        : VARCHAR         게시물 제목
  - content      : TEXT            게시물 본문
  - thumbnail_url: VARCHAR         썸네일 이미지 URL
  - view_count   : INTEGER         조회수
  - created_at   : TIMESTAMP       작성 시간
  - updated_at   : TIMESTAMP       수정 시간

[3] comments (댓글 테이블)
  - id         : UUID        PK  고유 식별자
  - post_id    : UUID        FK  게시물 (posts.id 참조)
  - user_id    : UUID        FK  작성자 (users.id 참조)
  - content    : TEXT            댓글 내용
  - created_at : TIMESTAMP       작성 시간
  - updated_at : TIMESTAMP       수정 시간

[4] likes (좋아요 테이블)
  - id         : UUID        PK  고유 식별자
  - post_id    : UUID        FK  게시물 (posts.id 참조)
  - user_id    : UUID        FK  좋아요 누른 유저 (users.id 참조)
  - created_at : TIMESTAMP       좋아요 누른 시간
  ※ (post_id + user_id) UNIQUE 제약 → 중복 좋아요 방지

테이블 관계:
  users ──┬── posts    (1:N)
          ├── comments (1:N)
          └── likes    (1:N)
  posts  ──┬── comments (1:N)
            └── likes    (1:N)

CRUD 매핑:
  회원가입          → users    CREATE
  닉네임 변경       → users    UPDATE
  회원탈퇴          → users    DELETE
  게시물 목록/상세  → posts    READ
  게시물 작성       → posts    CREATE
  게시물 수정/삭제  → posts    UPDATE / DELETE
  댓글 작성         → comments CREATE
  댓글 수정/삭제    → comments UPDATE / DELETE
  좋아요 누르기     → likes    CREATE
  좋아요 취소       → likes    DELETE
